Agis en tant que développeur Frontend Expert (React, TypeScript, Tailwind CSS). Ton objectif est de concevoir et configurer l'architecture d'un **Monorepo** et de développer l'interface utilisateur (UI) complète pour "ISTA-Goma PWA", un Système de Gestion Universitaire multi-portails.

IMPORTANT : Pour cette phase, il n'y a AUCUN backend ni base de données. Toute la donnée doit provenir d'un fichier statique `shared-data/data.json` consommé via des hooks locaux.

### 1. Structure de l'Architecture (Monorepo)
Le projet doit obligatoirement être structuré en **Monorepo (via pnpm workspaces ou npm workspaces)** sans Turbopack, utilisant **Vite** comme unique build tool pour toutes les applications :
- `packages/ui/` : Composants partagés (shadcn/ui, primitives).
- `packages/shared-data/` : Types TypeScript stricts, fichier `data.json` et hooks de simulation.
- `apps/auth/` : Portail de connexion centralisé.
- `apps/student/` : Application dédiée au portail étudiant.
- `apps/teacher/` : Application dédiée au portail enseignant.
- `apps/admin/` : Application regroupant les rôles administratifs (Apparitorat, Secrétariats, Rectorat).

### 2. Stack Technique Obligatoire
- **Framework & Build :** React 18+ avec TypeScript, propulsé exclusivement par **Vite** (Strictement **AUCUN Turbopack**).
- **Routage :** **TanStack Router** (architecture de routage de fichiers entièrement typée).
- **Styling :** Tailwind CSS 4 (utilisation stricte et exclusive via les variables globales).
- **Composants :** shadcn/ui (version latest, configuré pour un monorepo).
- **Icônes & Graphiques :** Lucide React & Recharts.

### 3. Principes de Design & UI/UX
- **Design System :** Utilise une esthétique ultra-fonctionnelle, monochrome et épurée (style Obsidian & Ivory). La palette doit s'appuyer rigoureusement sur les variables CSS standards (`--background`, `--foreground`, `--primary`, `--muted`, `--border`).
- **Performance visuelle :** Privilégie la clarté et la vitesse de lecture. **N'ajoute strictement aucune animation inutile, superflue ou décorative.**
- **Responsive :** Approche mobile-first obligatoire via les grilles et flexbox Tailwind.
- **Thème :** Support natif du Light/Dark mode via la classe `.dark` sur l'élément HTML.

### 4. Architecture des Données & Simulation Asynchrone
Génère un fichier `packages/shared-data/src/data.json` exhaustif contenant les données de test :
- `users` (6 rôles : Étudiant, Enseignant, Apparitorat, Secrétariat Faculté, Secrétariat Général, Rectorat).
- `students`, `teachers`, `courses`, `promotions`, `faculties`, `schedules`, `grades`, `announcements`.

Fournis le hook `usePageData.ts` (dans `packages/shared-data`) utilisant des promesses avec `setTimeout` pour simuler la latence réseau et tester les états de chargement dans les applications.

### 5. Routage & Authentification (Simulé via TanStack Router)
- **Gestion des accès :** Une connexion réussie stocke le rôle dans le `localStorage` (`ista-role`).
- Le portail d'authentification (`apps/auth`) affiche 6 cartes de connexion rapide (une par rôle). Au clic, l'utilisateur est redirigé vers l'application correspondante du monorepo.
- Chaque application utilise TanStack Router avec des routes protégées (`beforeLoad`) pour vérifier le rôle de l'utilisateur avant l'affichage.

### 6. Composants UI Communs à Développer (`packages/ui/src/components/`)
Crée les composants réutilisables et strictement typés suivants :
1. `PageHeader` : Reçoit `title`, `subtitle`, et un `action` (ReactNode).
2. `KPICard` : Reçoit `title`, `value`, `subtitle`, `icon`, et `colorClass`.
3. `StatusBadge` : Badge Tailwind coloré (vert, jaune, rouge) selon le statut (`active`, `pending`, `validated`).
4. `DataTable` : Tableau générique basé sur des colonnes configurables et un jeu de données.
5. `EmptyState` : État vide avec icône, titre et description.
6. `Loader` : Composant de chargement basé sur `Loader2` de lucide-react avec `animate-spin`.

### 7. Vues à Générer pour cette Itération
Fournis le code complet pour les pages suivantes :
1. **LoginPage (`apps/auth`) :** Écran de sélection visuelle des portails.
2. **StudentDashboard (`apps/student`) :** KPIs (Cours, Moyenne), grille d'emploi du temps (`ScheduleGrid`), dernières annonces.
3. **TeacherDashboard (`apps/teacher`) :** KPIs (Étudiants suivis, Notes à valider), planning du jour.
4. **Apparitorat Inscriptions (`apps/admin`) :** Route `/inscriptions` affichant la liste des étudiants via `DataTable` et un bouton ouvrant un `Dialog` shadcn/ui contenant le formulaire d'inscription complet.

### Règles de Livraison du Code
- Structure le code par blocs clairs en indiquant le chemin complet relatif au monorepo en commentaire (ex : `// apps/student/src/routes/index.lazy.tsx`).
- Le code doit être complet, fonctionnel et typé de manière stricte (pas de `any`). Crée un fichier de types partagé `packages/shared-data/src/types.ts`.
- Évite les raccourcis de type `// ... reste du code`.

Commence par générer l'arborescence des fichiers du monorepo, le fichier `types.ts` et le fichier `data.json`, puis enchaîne avec la configuration des routes TanStack et les composants UI.
